# Lab 4 — Maven

**OOP Java: Класи, ArrayList, Maven**

## Опис

Програма реалізує клас `Employee` (працівник) та драйвер-клас `Main`, який:
- зчитує з клавіатури кількість працівників;
- для кожного запитує ім'я та прізвище;
- зберігає всі об'єкти в `ArrayList<Employee>`;
- виводить інформацію через `toString()`;
- демонструє роботу методу `equals()`.

## Структура проєкту

```
employee-app/
├── pom.xml
├── .gitignore
├── README.md
└── src/
    └── main/
        └── java/
            └── com/
                └── lab1/
                    ├── Employee.java   ← клас предметної області
                    └── Main.java       ← драйвер (main)
```

## Поля класу Employee

| Поле              | Тип      | Опис               |
|-------------------|----------|--------------------|
| `firstName`       | `String` | Ім'я               |
| `lastName`        | `String` | Прізвище           |
| `position`        | `String` | Посада             |
| `salary`          | `double` | Зарплата (грн)     |
| `experienceYears` | `int`    | Стаж (роки)        |

## Збірка та запуск

```bash
# Збірка
mvn clean compile

# Запуск
mvn exec:java -Dexec.mainClass="com.lab.Main"
```

# Lab 5 — Винятки та Автотести (Maven)

**OOP Java: Валідація даних, Обробка винятків, JUnit 5, ArrayList**

## Опис

Програма реалізує клас `Employee` (працівник) з інкапсульованою бізнес-логікою та драйвер-клас `Main` у вигляді інтерактивного консольного меню.

**Ключові особливості цієї версії:**
- **Валідація даних:** Клас `Employee` перевіряє коректність вхідних даних (зарплата > 0, стаж >= 0, непусті імена) та викидає `IllegalArgumentException` при порушенні правил.
- **Обробка винятків (try-catch):** Драйвер-клас обробляє помилки вводу користувача (`InputMismatchException`) та помилки валідації, гарантуючи стабільну роботу програми без аварійних завершень.
- **Модульне тестування:** Перевірка логіки винятків автоматизована за допомогою **JUnit 5**.
- **Демонстрація equals():** Запобігання додаванню повних дублікатів працівників у список.
